# Sprout Track Documentation

Hi There,

Welcome to the documentation folder.  This is a collection of documents that have been put together to provide context and usage of Sprout Track.  Most of the documents are AI generated but reviewed by humans for accuracy with the exception of the User-Documentation folder which is made by hand.

**Admin-Documentation** - This is where Sprout Track administration documentation exists for self-hosters.  This is a more detailed version of the readme, and provides deeper insights into managing the application.

**Architecture-Documentation** - This is a 1.0 state of the current software architectural.  This is designed to help anyone who wants to take Sprout Track and morph it into their own, or use AI to assist in building within the app if you fork it.

**Implementation** - This is a historical archive of all mockups, prompts for expansive features, and planning documentation for building Sprout Track.  I figured I would include it if someone was interested in my approach to building this app.  All future implementation documentation will also live here.

**User-Documentation** - This is where all of the user facing documentation lives. Made by hand for end-users to use the app. Give this to your mother if you are asking her to use sprout track while taking care of your little one. Once this is complete this "manual" will be available in app too. It might even be fully localized (well it better be)

There are also some random .csv's. Don't worry about that. It's CDC percentile data for height, weight, and head circumference.
